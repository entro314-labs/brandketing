# Generate Skill - Strategy Workflow Reference

## The 6-Step Workflow

This is the complete strategy workflow. The strategist agent follows these steps in order.

### Step 1: Product Truth Extraction

Turn PRODUCT EVIDENCE into a structured table:

| Capability | User action | Persona | Trigger moment | Friction removed | Proof |
|---|---|---|---|---|---|

**Rule:** Code and docs confirm existence. To become message-worthy, a capability must also be supported by: users describing the related pain, analytics showing it affects value, competitors being weak there, or proof making it credible.

### Step 2: Market Meaning Translation

Run each capability through the feature ladder:

```
Feature
  → Capability (what the user can now do)
  → Friction removed (what stops being slow, risky, confusing, manual, or fragile)
  → Outcome (what gets better in the user's work or life)
  → Second-order outcome (what that improvement unlocks next)
  → Meaning (why that matters emotionally, socially, or economically)
  → Positioning (why this is more worth caring about than adjacent features or competitor claims)
```

Assign each capability to: hero / subhead / pillar / proof / omit

### Step 3: Territory Generation

Generate 3–5 positioning territories. For each:

- Target audience
- Surface problem (what they think they have)
- Deeper problem (what they actually have - the latent diagnosis)
- Cost of misdiagnosis
- The promise
- Unique mechanism making the promise believable
- Why now
- Competitor overlap
- Available proof
- Territory risks

### Step 4: Territory Scoring

Score each territory 1–5 on:
- Pain intensity
- Frequency
- Differentiation
- Proof strength
- Emotional resonance
- Speed to value
- Strategic fit for the product

Subtract if: generic, verbatim competitor claim, feature-name dressed up, hype without proof.

**Pause for human to confirm primary territory.**

### Step 5: Messaging Hierarchy

Write for the confirmed primary territory only:

- **Core brand message** - the changed reality for the buyer. Not the feature, not the architecture. The shift.
- **5 headline options** - outcome-led, tension-driven, specific
- **3 subhead options** - mechanism + relevance + who it's for
- **3 message pillars** - each maps a product strength to a meaningful benefit
  - Support bullets under each pillar (translated features, not feature names)
- **3 CTA options** - action-specific, not generic
- **3 tagline options** - short, ownable, preferably creates tension or reframe

Hierarchy rule:
- Hero = outcome
- Subhead = mechanism + relevance
- Bullets = translated features
- Proof = specifics

### Step 6: Self-Critique

Audit every line for:
- Feature-led instead of outcome-led
- Generic SaaS language
- Arrogant or overclaimed
- Clowny or too clever
- Vague
- Not rooted in audience language
- Sounds like paraphrased docs

Rewrite weak lines. Do not mark lines clean that aren't.

## Context Weighting Rule

Same feature, different ICP = different message:

| ICP | Frame |
|---|---|
| Individual user | Personal productivity and friction removal |
| Team lead | Rollout speed and team adoption |
| Admin / ops buyer | Control, standardization, no firefighting |

Apply ICP framing throughout all 5 steps.

## Banned Words (without proof)

seamless, robust, powerful, intuitive, innovative, next-gen, best-in-class, revolutionary, game-changing, state-of-the-art, cutting-edge, holistic, synergy, leverage (as verb), utilize
